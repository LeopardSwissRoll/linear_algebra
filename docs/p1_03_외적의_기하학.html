PHASE 1

외적(Cross Product)의 기하학

두 벡터가 얼마나 "다른 방향"인가

내적의 쌍둥이, 그런데 전혀 다른

내적이 "얼마나 같은 방향인가"를 재는 도구였다면, 외적은 "얼마나 다른 방향인가"를 잰다.

결과도 다르다. 내적은 스칼라(숫자 하나)를 돌려주지만, 외적은 새로운 벡터를 돌려준다 — 두 벡터에 모두 수직인 방향, 크기는 두 벡터가 만드는 평행사변형의 넓이.

$$|\vec{a} \times \vec{b}| \;=\; |\vec{a}|\,|\vec{b}|\sin\theta \;=\; \text{평행사변형 넓이}$$

2D에서는 z-성분만 남는다: $a_x b_y - a_y b_x$ (부호 있는 넓이)

3D에서는 — 법선 벡터

2D에서 외적은 숫자 하나(부호 있는 넓이)를 줬다. 3D에서는 벡터를 준다 — 두 벡터가 만드는 평면에 수직인 방향이다. 이것을 법선 벡터(normal vector)라고 부른다.

아래에서 궤도를 돌려보면, $\vec{a}$와 $\vec{b}$가 만드는 평면(바닥)에서 외적 $\vec{c}$가 수직으로 솟아오르는 것을 볼 수 있다.

오른손 법칙 — 그리고 DirectX는 왼손

오른손의 네 손가락을 $\vec{a}$에서 $\vec{b}$ 쪽으로 감아쥐면, 엄지가 가리키는 방향이 $\vec{a} \times \vec{b}$의 방향이다.

주의: OpenGL은 오른손 좌표계, DirectX는 왼손 좌표계를 쓴다. 왼손 좌표계에서는 와인딩 규약이 뒤집히므로, 앞면을 CW로 정의한다. Phase 3에서 다시 만날 내용이다.

외적은 3차원 전용 도구다

"두 벡터에 수직인 벡터"라는 개념은 3D에서만 유일하게 정의된다. 2D에서는 수직 방향이 하나뿐이라 스칼라로 충분하고, 4D 이상에서는 수직 방향이 여러 개라 외적이 성립하지 않는다. 일반 선형대수의 중심은 내적·정사영·기저·직교성이고, 외적은 3D 기하학/그래픽스의 전용 무기다.

그래서 외적이 어디 쓰이는데?

삼각형 법선 계산: 세 꼭짓점 A, B, C → 변 벡터 (B−A)×(C−A) = 표면 법선. 모든 3D 렌더러가 조명 계산에 이것을 쓴다. Phase 1 과제 1-3에서 직접 짠다.

뒷면 판별: 삼각형의 와인딩 순서(CW/CCW)를 외적의 부호로 판별 → 카메라를 등지는 삼각형을 그리지 않는 back-face culling의 핵심이다.

직접 해보기

  1. 넓이 = 0: 두 벡터를 평행하게 만들어봐. 평행사변형이 찌그러지면서 넓이가 0이 되는 걸 확인.
  2. 최대 넓이: 같은 크기의 두 벡터로 넓이를 최대로 만들어봐. 어떤 각도일 때? (sin θ를 생각해봐.)
  3. 와인딩 뒤집기: $\vec{b}$를 $\vec{a}$의 반대편으로 넘겨봐. 부호가 뒤집히고 색이 바뀌는 걸 확인.
  4. 3D 궤도: 궤도 슬라이더를 돌려서 법선 벡터가 정말로 바닥 평면에 수직인지 확인.
