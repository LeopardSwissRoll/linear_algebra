PHASE 3

Model 변환 (로컬 → 월드)

오브젝트를 세계에 배치하기. SRT 순서와 계층 구조.

공간(Space)이란?

로컬 공간 (Local)

오브젝트가 원점에 놓인 상태. 모델러가 만든 그대로.

월드 공간 (World)

게임 세계의 절대 좌표계. 모든 오브젝트가 공유하는 기준.

$$\vec{v}_{world} = M_{model} \cdot \vec{v}_{local} \qquad M_{model} = T \cdot R \cdot S$$

여러 오브젝트 배치 — 2D 미니 씬

오브젝트를 클릭해서 선택하고, SRT 슬라이더로 배치해봐. 각 오브젝트는 자신만의 Model 행렬을 가진다.

Sx1.00
Sy1.00
R0°
Tx2.0
Ty3.0

선택된 오브젝트의 Model 행렬

M = T · R · S
월드 위치 (4열)

계층 구조 — 2D 로봇팔

$$M_{arm\_world} = M_{body} \cdot M_{arm\_local}$$

부모가 움직이면 자식도 같이 움직인다. 게임 캐릭터의 관절 시스템의 본질.

어깨 30°
팔꿈치 45°

관절 행렬

윗팔 (어깨): Mupper
아래팔 (로컬): Mlower_local
아래팔 (월드): Mupper · Mlower
부모의 변환이 자식에게 전파된다

어깨를 돌리면 팔꿈치도 같이 움직인다 — $M_{body}$가 바뀌니까. 팔꿈치를 돌리면 아래팔만 움직인다 — $M_{arm\_local}$만 바뀌니까. 이게 스켈레탈 애니메이션의 기본.

비균등 스케일과 법선 — 주의!

미니 씬에서 $S_x \neq S_y$로 놓아봐. 오브젝트 형태가 찌그러진다. 이때 법선 벡터를 같은 Model 행렬로 변환하면 표면에 수직이 아니게 된다 — 조명이 깨지는 원인. 해결법은 $(M^{-1})^T$ (역전치). 상세는 p3_05에서.

직접 해보기

  1. SRT 조합: 삼각형 A의 스케일을 (2, 0.5), 회전 60°, 이동 (4, 1)로 설정. 결과 확인.
  2. 다중 오브젝트: 세 오브젝트를 각각 다른 위치/각도로 배치. 각 Model 행렬이 다른 걸 확인.
  3. 계층 관찰: 어깨만 돌려봐. 아래팔도 같이 움직이는 이유는?
  4. 로컬 축: "로컬 축 보기"를 켜고 회전해봐. 로컬 축이 오브젝트와 함께 돌아가는 걸 확인.
  5. 행렬의 4열: Model 행렬의 마지막 열(이동 부분)이 오브젝트의 월드 위치와 같은 이유는?
