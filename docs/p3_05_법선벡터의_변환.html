법선 벡터의 변환

법선에 같은 행렬을 곱하면 안 된다 — 비균등 스케일 하나면 깨진다

문제 — 법선이 깨진다

법선 = 표면에 수직인 벡터. 조명의 핵심: $\text{밝기} = \max(0,\;\vec{n} \cdot \vec{L})$. 법선이 틀리면 조명이 깨진다.

균등 스케일 → OK

모든 방향을 같은 비율로 늘리면, 수직 관계가 유지된다.

비균등 스케일 → 깨짐!

x만 2배로 늘리면 법선도 x 방향으로 2배 → 더 이상 수직이 아님.

법선 비교

M으로 변환한 법선 (빨강 — 잘못됨)
(M⁻¹)ᵀ로 변환한 법선 (초록 — 올바름)
접선과의 내적 (0이면 수직)

왜 $(M^{-1})^T$인가

접선 $\vec{t}$에 수직인 관계를 보존해야 한다

접선: $\vec{t'} = M\vec{t}$,   법선: $\vec{n'} = ?\vec{n}$

수직 조건: $\vec{n'}^T \vec{t'} = 0$

$\vec{n'}^T (M\vec{t}) = (\vec{n'}^T M)\vec{t} = 0$

이것이 $\vec{n}^T \vec{t} = 0$과 같으려면:

$$\boxed{\vec{n'} = (M^{-1})^T \vec{n}}$$

4줄 유도. 스케일이 커진 방향의 법선은 오히려 줄어야 수직이 유지된다.

특수 케이스

변환$(M^{-1})^T$법선도 $M$으로 OK?
균등 스케일 $sI$$\frac{1}{s}I$O (정규화하면 같음)
회전 $R$$R$ 자체O (직교행렬)
비균등 스케일$\text{diag}(1/s_x, 1/s_y, 1/s_z)$X — 깨짐!
실전 팁

직교 행렬(회전, 반사)이면 $M^{-1} = M^T$이므로 $(M^{-1})^T = M$ — 같은 행렬로 OK. 비균등 스케일이 포함된 경우만 역전치가 필요.

조명 Before/After

타원 표면의 여러 점에서 법선을 구하고, Lambert 조명을 적용. 광원 방향을 드래그해봐.

Lambert 조명

광원 방향 L
밝기 = max(0, n · L)

실전: 셰이더 코드

// 정점 셰이더 (DX11 HLSL)
float3 worldNormal = mul((float3x3)transpose(invWorld), input.normal);
worldNormal = normalize(worldNormal);

invWorld = $M^{-1}$. transpose = $(M^{-1})^T$. normalize = 스케일 영향 제거.

최적화

균등 스케일만 쓰면 mul((float3x3)world, normal)로 충분. CPU에서 역전치를 미리 계산해서 상수 버퍼로 전달하는 게 일반적.

직접 해보기

  1. 깨지는 확인: 캔버스 ①에서 Sx=3, Sy=1로 놓고 빨간 법선이 표면에 수직인지 확인.
  2. 균등이면 OK: Sx=Sy=2로 놓으면 빨간/초록 법선이 겹치는 걸 확인.
  3. 조명 비교: 캔버스 ②에서 "잘못된 법선" 토글. 어디가 가장 차이 나는지?
  4. 유도 복습: $\vec{n'}^T M \vec{t} = \vec{n}^T \vec{t}$에서 왜 $(M^{-1})^T$가 나오는지 적어봐.
  5. HLSL: transpose(invWorld)가 $(M^{-1})^T$인 이유를 코드와 수식으로 대응시켜봐.
