## Phase 1 — 확률적 사고의 재정립 (1주)

"개념이 뭔진 안다"에서 "직관적으로 쓸 수 있다"로 올리는 단계.
공식을 다시 외우는 게 아니라, **확률을 기하학적·시뮬레이션적으로 보는 관점**을 잡는다.

### 핵심 개념

**확률분포 = 공간 위의 질량 분배**
- 이산: 확률질량함수(PMF) — 각 점에 질량을 올려놓은 것
- 연속: 확률밀도함수(PDF) — 질량이 면적으로 퍼진 것
- PDF의 값 자체는 확률이 아님. 구간의 적분이 확률
- 고차원으로 가면 "질량이 어디에 몰려 있는가"가 핵심 질문이 됨

**기댓값 = 질량 중심, 분산 = 퍼짐의 정도**
- 기댓값: 분포의 무게중심. 선형성이 가장 강력한 성질 (E[aX+bY] = aE[X]+bE[Y], 독립 불필요)
- 분산: 중심으로부터의 평균 제곱 거리. Var(X) = E[X²] - (E[X])²
- 표준편차 vs 분산: 단위가 다르다. 분산은 제곱 단위

**결합 분포와 주변 분포**
- 두 변수의 결합 분포 p(x,y)에서 한 변수를 적분하면 주변 분포
- 조건부 분포 p(x|y) = p(x,y)/p(y) — 슬라이스를 떠서 정규화한 것
- 기하학적으로: 결합 분포의 단면

**빈도주의 vs 베이지안 — 관점의 차이**
- 빈도주의: 확률 = 반복 실험의 상대 빈도. 파라미터는 고정된 미지수
- 베이지안: 확률 = 불확실성의 정도. 파라미터도 확률변수
- 실용적 차이: 베이지안은 사전 지식을 공식적으로 반영할 수 있음
- 센서 퓨전에서 왜 베이지안이 자연스러운가: 이전 측정(사전)과 새 측정(가능도)을 합치는 것

## Phase 2 — 베이즈 정리와 베이지안 추론 (2주)

센서 퓨전, 칼만 필터, MDN 전부의 기반. "베이즈 정리를 안다"에서 "자연스럽게 쓴다"로 가는 단계.

### 핵심 개념

**베이즈 정리의 해부**
```
posterior ∝ likelihood × prior

p(θ|D) = p(D|θ) · p(θ) / p(D)
```
- prior p(θ): 데이터를 보기 전의 믿음
- likelihood p(D|θ): "이 파라미터가 참이라면 데이터가 나올 확률"
- posterior p(θ|D): 데이터를 본 후의 업데이트된 믿음
- evidence p(D): 정규화 상수. 계산이 어려운 것이 베이지안의 핵심 난제

**순차 업데이트 (Sequential Update)**
- 데이터가 하나씩 들어올 때, 이전의 posterior가 다음의 prior가 됨
- 센서 퓨전의 본질: 측정이 올 때마다 posterior를 갱신
- 칼만 필터가 이것을 가우시안 가정 하에서 닫힌 형태로 해결한 것

**켤레 사전분포 (Conjugate Priors)**
- prior와 posterior가 같은 분포 가족에 속하면 계산이 닫힌 형태로 풀림
- Beta-Binomial: 동전의 앞면 확률 추정
- Normal-Normal: 평균 추정
- 왜 중요한가: 닫힌 형태가 아니면 적분을 풀 수 없어서 MCMC 등이 필요

**MAP vs 완전 베이지안**
- MAP (Maximum A Posteriori): posterior의 최빗값만 취함. 점추정
- 완전 베이지안: posterior 분포 전체를 유지. 불확실성까지 표현
- ML의 정규화(L2)가 사실은 가우시안 prior를 둔 MAP와 동일

## Phase 3 — 가우시안의 모든 것과 혼합 모델 (2주)

가우시안이 왜 특별한지, GMM과 EM이 왜 중요한지를 잡는 단계. 센서 퓨전과 MDN의 수학적 기반.

### 핵심 개념

**다변량 가우시안의 기하학**
- 평균 벡터 μ: 분포의 중심
- 공분산 행렬 Σ: 타원체의 형태와 방향을 결정
- Σ의 고유값 분해 = 타원의 주축 방향과 길이 (선형대수 Phase 5와 직결!)
- 마할라노비스 거리: Σ를 고려한 "진짜" 거리. (x-μ)ᵀ Σ⁻¹ (x-μ)

**가우시안이 특별한 이유들**
- 중심극한정리: 무엇을 더하든 가우시안으로 수렴
- 가우시안의 곱 = 가우시안 (베이지안 업데이트가 닫힌 형태)
- 가우시안의 선형 변환 = 가우시안 (칼만 필터가 가능한 이유)
- 최대 엔트로피: 평균과 분산만 알 때 가장 가정이 적은 분포
- 이 성질들이 없으면 칼만 필터, 센서 퓨전, 정규 분포 가정 ML 전부 불가능

**가우시안 혼합 모델 (GMM)**
- 문제: 세상의 데이터는 하나의 가우시안으로 설명 안 됨
- 해법: K개의 가우시안을 가중합으로 섞기
- p(x) = Σ πₖ N(x|μₖ, Σₖ), 여기서 Σπₖ = 1
- 의미: "데이터가 K개의 클러스터 중 하나에서 왔다"

**EM 알고리즘**
- 문제: GMM의 파라미터(μₖ, Σₖ, πₖ)를 어떻게 추정하는가?
- 닭-달걀 문제: 소속을 알면 파라미터를 추정할 수 있고, 파라미터를 알면 소속을 추정할 수 있다
- E-step: 현재 파라미터로 각 데이터의 소속 확률(responsibility) 계산
- M-step: 소속 확률로 가중 평균하여 파라미터 업데이트
- 수렴 보장: 매 iteration마다 log-likelihood가 증가 (로컬 최적은 가능)

## Phase 4 — 노이즈 함수와 프로시저럴 생성 (2주)

확률을 게임 그래픽스에 직접 적용하는 단계. UE의 머티리얼 에디터에서 매일 쓰이는 수학.

### 핵심 개념

**난수 vs 노이즈: 결정적 차이**
- 순수 난수: 인접한 값 사이에 상관관계 없음 → 화면에 뿌리면 TV 정적(static)
- 노이즈 함수: 공간적으로 연속이고 부드러움 → 자연스러운 패턴
- 핵심: 노이즈 = "공간적 상관관계가 있는 의사난수 함수"

**Perlin Noise의 수학적 구조**
- 격자점마다 랜덤 그래디언트 벡터 배치
- 격자 내부: 각 격자점의 그래디언트와 오프셋 벡터의 내적 계산
- 보간: fade 함수(6t⁵ - 15t⁴ + 10t³)로 부드럽게 섞음
- 이 fade 함수가 왜 이 형태인가: 1차·2차 도함수가 격자점에서 0 → C² 연속

**Simplex Noise (개선판)**
- Perlin의 문제: 고차원에서 계산량 기하급수적 증가 (2ⁿ 꼭짓점)
- Simplex: n차원에서 n+1 꼭짓점만 사용 (삼각형 격자 기반)
- 방향성 아티팩트가 적고, 계산이 빠름

**fBm (Fractal Brownian Motion)**
- 노이즈를 여러 옥타브(octave)로 쌓아올리기
- 각 옥타브: 주파수 2배, 진폭 절반 (lacunarity, gain 파라미터)
- 결과: 프랙탈 같은 자기유사성 → 지형, 구름, 대리석 질감
- 옥타브 수를 조절하면 디테일 수준 제어 가능

**도메인 워핑 (Domain Warping)**
- 노이즈 입력에 다른 노이즈를 넣기: f(p + noise(p))
- 소용돌이, 연기, 유체 같은 유기적 패턴 생성
- 다중 워핑으로 복잡도 증가

**게임에서의 프로시저럴 생성 패턴**
- 지형 생성: 높이맵 = fBm, 침식 시뮬레이션
- 텍스처: 노이즈 조합으로 나무결, 돌, 구름 패턴
- 동굴/던전: 셀룰러 오토마타 + 노이즈 threshold
- UE 머티리얼 에디터에서의 노이즈 노드 활용

## Phase 5 — 상태 공간 모델과 칼만 필터 (2~3주)

센서 퓨전의 수학적 핵심. IMU+UWB 경험이 있으니 동기 부여는 이미 되어 있을 것.
여기서 목표는 "칼만 필터를 쓸 줄 안다"가 아니라 **"왜 이 구조인지를 유도할 수 있다"**.

### 핵심 개념

**상태 공간 모델의 구조**
```
상태 전이:   xₜ = Fxₜ₋₁ + Buₜ + wₜ     (wₜ ~ N(0, Q))
관측 모델:   zₜ = Hxₜ + vₜ              (vₜ ~ N(0, R))
```
- 상태 x: 우리가 추정하고 싶은 것 (위치, 속도, 자세 등)
- 관측 z: 센서가 실제로 주는 것 (노이즈 포함)
- F: 시스템이 시간에 따라 어떻게 변하는가 (물리 모델)
- H: 상태에서 관측이 어떻게 만들어지는가 (센서 모델)
- Q, R: 프로세스 노이즈와 관측 노이즈의 공분산

**칼만 필터의 두 단계**
- Predict: 이전 posterior + 시스템 모델 → prior 예측
  - μ⁻ = Fμ + Bu
  - Σ⁻ = FΣFᵀ + Q
- Update: prior + 새 관측 → posterior 업데이트
  - K = Σ⁻Hᵀ(HΣ⁻Hᵀ + R)⁻¹ (칼만 게인)
  - μ = μ⁻ + K(z - Hμ⁻)
  - Σ = (I - KH)Σ⁻
- 핵심 직관: 칼만 게인 K는 "예측을 얼마나 믿을까 vs 관측을 얼마나 믿을까"의 저울

**가우시안 가정의 역할**
- predict: 가우시안의 선형 변환 = 가우시안 → 닫힌 형태
- update: 가우시안의 곱 = 가우시안 → 닫힌 형태
- 이 두 성질이 없으면 칼만 필터의 우아한 재귀 구조가 무너짐
- Phase 3에서 배운 "가우시안이 특별한 이유"가 여기서 전부 쓰임

**확장 칼만 필터 (EKF)**
- 문제: 실제 시스템은 비선형 (예: 회전이 포함된 운동)
- 해법: 비선형 함수를 현재 추정값에서 선형화 (야코비안!)
- 선형대수 Phase 6에서 배운 야코비안이 여기서 직접 쓰임
- 한계: 강한 비선형에서 선형화 오차가 누적

**무향 칼만 필터 (UKF)**
- EKF의 대안: 야코비안 대신 "시그마 포인트"를 비선형 함수에 통과
- 평균과 공분산을 시그마 포인트들의 가중 평균으로 복원
- 야코비안을 유도할 필요 없음, 비선형에서 더 정확
- 계산량은 EKF보다 약간 많지만 구현이 더 단순한 경우가 많음

**센서 퓨전으로의 연결**
- IMU: 고빈도, 드리프트 있음 → 프로세스 모델로 사용
- UWB/GPS: 저빈도, 절대 위치 → 관측 모델로 사용
- 칼만 필터가 둘을 최적으로 합치는 프레임워크
- NLOS 오차 → 관측 노이즈 R을 적응적으로 조절하는 문제

## Phase 6 — MDN과 확률적 심층 모델 (2주)

확률 + 딥러닝의 교차점. 모션 예측, 포즈 추정에 직결.

### 핵심 개념

**일반 신경망의 한계: 점추정**
- 표준 회귀 신경망: 입력 x → 출력 ŷ (하나의 값)
- 문제: 같은 입력에 대해 여러 출력이 가능한 경우? (다대다 매핑)
- 예시: 부분 관측된 포즈 → 가능한 전체 포즈가 여러 개
- 점추정으로는 다중 모드를 표현할 수 없음 (평균으로 뭉개짐)

**Mixture Density Network (MDN)**
- 출력을 GMM의 파라미터(πₖ, μₖ, σₖ)로 변환
- p(y|x) = Σ πₖ(x) · N(y|μₖ(x), σₖ²(x))
- 신경망이 "하나의 답"이 아니라 "답의 분포"를 출력
- 입력에 따라 분포의 형태 자체가 달라짐

**학습**
- 손실 함수: negative log-likelihood
- L = -log Σ πₖ N(y|μₖ, σₖ²)
- 안정적 계산: log-sum-exp 트릭
- 수치 안정성: σ에 softplus, π에 softmax

**모션 예측에서의 MDN**
- 입력: 과거 관절 위치/속도 시퀀스
- 출력: 다음 프레임의 관절 위치의 확률 분포
- 불확실성이 큰 시점(분기점)에서 여러 모드가 나타남
- 모캡 데이터의 다중 모드 특성을 자연스럽게 처리

**조건부 VAE / Diffusion (소개 수준)**
- MDN의 확장: 더 복잡한 분포를 모델링하는 방향
- VAE: 잠재 공간의 분포를 명시적으로 모델링
- Diffusion: 노이즈에서 점진적으로 데이터를 복원
- 이것들도 결국 "확률 분포를 출력하는 신경망"의 변형

## Phase 7 — 파티클 필터와 몬테카를로 방법 (1~2주, 심화)

가우시안 가정이 깨질 때의 대안. Phase 5의 칼만 필터가 다루지 못하는 영역.

### 핵심 개념

**칼만 필터의 한계**
- 가우시안 가정: 실제 posterior가 다봉(multi-modal)이면 실패
- 선형 가정: EKF의 선형화가 부정확하면 발산 가능
- 예시: 로봇이 복도에 있는데 어느 복도인지 모를 때 → 다봉 분포

**파티클 필터 (Sequential Monte Carlo)**
- 아이디어: posterior를 가우시안 대신 "샘플(파티클)의 집합"으로 근사
- 각 파티클 = 하나의 가능한 상태, 각 파티클의 가중치 = 그 상태의 확률
- Predict: 각 파티클을 시스템 모델 + 노이즈로 전파
- Update: 관측에 대한 likelihood로 가중치 업데이트
- Resample: 가중치에 비례하여 파티클 재추출 (퇴화 방지)

**중요도 샘플링 (Importance Sampling)**
- 파티클 필터의 수학적 기반
- 원래 분포에서 직접 샘플링이 어려울 때, 다른 분포에서 샘플링하고 가중치로 보정
- 파티클의 가중치 = likelihood / proposal

**MCMC (마르코프 체인 몬테카를로) — 소개 수준**
- posterior에서 직접 샘플링하는 일반적 방법
- Metropolis-Hastings: 제안 → 수락/거부의 반복
- 베이지안 추론에서 evidence p(D)를 직접 계산하지 않고도 posterior 샘플 가능
- 게임에서의 응용: 복잡한 분포에서의 샘플링, 전역 조명(path tracing)의 수학적 기반
