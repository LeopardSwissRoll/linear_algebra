## Phase 1 — 벡터와 내적·외적의 기하학 (1주)

이미 벡터 연산 자체는 알고 있을 것. 여기서 목표는 "계산"이 아니라 **기하학적 문장으로 읽는 능력**.

### 핵심 개념

**벡터의 이중 정체성**
- 점(position)과 방향(direction)은 같은 데이터지만 의미가 다르다
- 이 구분이 나중에 동차 좌표에서 w=1 vs w=0으로 정확히 대응됨

**내적 (dot product)**
- 공식보다 먼저: "한 벡터를 다른 벡터 위에 투영한 길이"
- cos θ와의 관계 → 두 벡터의 유사도 측정기
- 부호 판별: 같은 쪽이면 양, 직교면 0, 반대쪽이면 음

**외적 (cross product)**
- 두 벡터가 만드는 평면의 법선
- 크기 = 평행사변형의 넓이 → 두 벡터가 얼마나 "다른 방향"인가의 측도
- 오른손 법칙과 좌표계 규약 (DirectX는 왼손 좌표계)

**기저(basis)와 좌표계**
- 좌표 = "이 기저 벡터들을 각각 얼마나 쓰는가"
- 기저가 바뀌면 같은 점의 좌표가 달라진다 → 이것이 좌표 변환의 본질

### 구현 과제

```
과제 1-1: 순수 C++로 Vec3 구조체 구현
         - 덧셈, 스칼라곱, dot, cross, normalize, length
         - 연산자 오버로딩 포함
         - 단위 테스트: 알려진 결과와 비교

과제 1-2: 3D 공간에서 Lambert 조명 계산
         - 표면 법선 n과 광원 방향 L의 내적으로 밝기 결정
         - 콘솔 출력이어도 좋고, 간단한 픽셀 버퍼에 그려도 좋음
         - "내적이 음수면 그림자"를 직접 확인

과제 1-3: 삼각형의 법선 벡터 계산
         - 세 꼭짓점 → 두 변 벡터 → 외적 → 정규화
         - 와인딩 오더(CW vs CCW)에 따라 법선 방향이 뒤집히는 것 확인
```

## Phase 2 — 행렬 = 선형 변환 (2주)

가장 중요한 구간. 행렬을 "숫자 격자"가 아니라 **"공간을 변형시키는 함수"**로 보는 시각을 확립하는 단계.

### 핵심 개념

**행렬의 열(column) = 변환된 기저 벡터**
- 2×2 행렬 [a c; b d]는 "e₁이 (a,b)로, e₂가 (c,d)로 간다"는 뜻
- 이 하나의 관점만 확실히 잡으면 나머지가 전부 따라옴
- 행렬을 보는 즉시 "기저가 어디로 갔는가"를 떠올리는 훈련

**기본 변환들의 행렬 표현 (3D)**
- 스케일: 대각 행렬. 각 축을 늘리거나 줄임
- 회전: 직교 행렬. 기저 벡터의 길이 보존, 직교성 보존
- 반사: 행렬식 = -1. 좌표계의 손잡이가 뒤집힘
- 전단(shear): 한 축이 다른 축에 의존해서 밀림

**행렬 곱셈 = 변환의 합성**
- AB는 "B를 먼저 적용하고 A를 적용" (오른쪽에서 왼쪽)
- 교환법칙이 성립하지 않는 이유: 회전 후 스케일 ≠ 스케일 후 회전

**역행렬 = 변환의 되돌리기**
- 기하학적으로: 공간을 원래대로 펴는 변환
- 역행렬이 없다 = 차원이 찌그러졌다 (정보 손실)

**행렬식(determinant) = 부피 변화율**
- det > 0: 방향 보존, det < 0: 뒤집힘, det = 0: 차원 붕괴
- 직교 행렬의 det = ±1 (부피 보존)

### 구현 과제

```

         - 라이브러리 금지

과제 2-2: 변환 합성 실험
         - 같은 오브젝트에 (회전 → 스케일)과 (스케일 → 회전)을 적용
         - 결과 행렬을 출력하고, 변환된 꼭짓점 좌표를 비교
         - "순서가 왜 중요한가"를 숫자로 확인

과제 2-3: 소프트웨어 래스터라이저 (단순 버전)

```

## Phase 3 — 동차 좌표와 그래픽스 파이프라인 (2주)

DirectX 11과 직접 연결되는 단계. Phase 2에서 만든 수학 라이브러리 위에 그래픽스 파이프라인을 올린다.

### 핵심 개념

**동차 좌표계 (Homogeneous Coordinates)**
- 문제: 3×3 행렬로는 이동(translation)을 표현할 수 없다 (선형 변환이 아니니까)
- 해법: 차원을 하나 올려서 4×4로 만들면, 이동도 행렬 곱으로 표현 가능
- w 좌표의 의미: w=1이면 점, w=0이면 방향 벡터
- 아핀 변환을 선형 변환으로 임베딩하는 것

**MVP 변환 체인**
- Model: 오브젝트 로컬 공간 → 월드 공간
- View: 월드 공간 → 카메라 공간 (카메라를 원점으로 옮기는 것)
- Projection: 카메라 공간 → 클립 공간 (원근감 적용)
- 각 단계가 독립적인 4×4 행렬, 합치면 하나의 행렬

**원근 투영의 유도**
- 절두체(frustum)를 정육면체(NDC)로 찌그러뜨리는 변환
- near/far plane, FOV, aspect ratio가 행렬의 각 원소에 어떻게 들어가는지
- w 나누기(perspective divide)의 기하학적 의미: 먼 것이 작아지는 원리
- DirectX의 NDC z 범위: [0, 1]

**법선 벡터의 변환**
- 왜 법선은 같은 행렬로 변환하면 안 되는가 (비균등 스케일 반례)
- 정답: (M⁻¹)ᵀ — 역전치 행렬
- 직관: 법선은 "표면에 수직"이라는 관계를 보존해야 함

### 구현 과제

```
과제 3-1: MVP 행렬 직접 구성
         - LookAt 행렬을 직접 유도하고 구현
         - 원근 투영 행렬을 FOV, aspect, near, far로부터 직접 유도
         - DirectXMath의 결과와 비교 검증

과제 3-2: DirectX 11에서 최소 파이프라인 구축
         - 정점 셰이더에서 MVP 변환 적용
         - 카메라를 마우스로 회전시키면서 View 행렬 변화 관찰
         - 여기서부터 DirectXMath 사용 허용

과제 3-3: 법선 변환 버그 재현
         - 오브젝트에 비균등 스케일(예: x만 3배) 적용
         - 법선을 월드 행렬로 변환 → 조명이 깨지는 것 확인
         - 역전치로 교체 → 정상으로 돌아오는 것 확인
```

- "w=1이면 점, w=0이면 방향"인 이유를 설명할 수 있는가?

## Phase 4 — 회전의 깊은 이해 (2주)

회전은 선형대수에서 독립 토픽으로 다룰 만큼 깊고, 모캡/포즈 추정과 직결된다.

### 핵심 개념

**오일러 각의 한계**
- Roll-Pitch-Yaw: 직관적이지만 순서 의존적
- 짐벌 락(Gimbal Lock): 두 축이 정렬되면 자유도 하나를 잃음
- 보간이 부자연스러움 (선형 보간하면 경로가 휘어짐)

**회전 행렬과 SO(3)**
- 직교 행렬 중 det=+1인 것들의 집합
- 3개의 자유도를 9개의 숫자로 표현 → 중복이 많고 정규화 필요
- 장점: 변환 적용이 단순한 행렬 곱

**쿼터니언**
- q = w + xi + yj + zk — 4차원 복소수의 확장
- 단위 쿼터니언 = 3D 회전 (축-각 표현: q = cos(θ/2) + sin(θ/2)·axis)
- 곱셈 = 회전의 합성, 켤레 = 역회전
- Slerp: 구면 위의 선형 보간 → 일정한 각속도로 부드러운 회전
- 왜 θ/2인가? → 이중 피복(double cover) 개념

**듀얼 쿼터니언 (심화)**
- 회전 + 이동을 하나로 표현
- 스켈레탈 애니메이션에서 행렬 블렌딩 대비 스키닝 품질 향상
- "candy wrapper" 아티팩트 해결

### 구현 과제

```
과제 4-1: 짐벌 락 시각화
         - 오일러 각으로 회전을 제어하는 DX11 뷰어
         - Pitch를 ±90°로 가져갔을 때 조작이 깨지는 것 직접 체험

과제 4-2: 쿼터니언 라이브러리 직접 구현
         - 곱셈, 켤레, 역, 정규화
         - 쿼터니언 ↔ 행렬 변환
         - Slerp 구현 및 오일러 각 Lerp와 비교

과제 4-3: 스켈레탈 애니메이션 블렌딩
         - 두 포즈 사이를 쿼터니언 Slerp으로 보간
         - 같은 작업을 행렬 Lerp로 했을 때의 차이 확인
         - 모캡 데이터가 있으면 직접 활용
```

## Phase 5 — 고유값 분해와 SVD (2주)

ML 쪽과의 연결이 본격적으로 시작되는 단계.

### 핵심 개념

**고유벡터·고유값의 기하학적 의미**
- 변환해도 방향이 안 바뀌는 벡터 = 고유벡터
- 그 벡터가 늘어나거나 줄어드는 비율 = 고유값
- 변환의 "본질적인 축"을 드러내는 것

**고유값 분해 (Eigendecomposition)**
- A = PDP⁻¹ — 변환을 "고유 기저로 가서, 스케일하고, 돌아오기"로 분해
- 대칭 행렬이면 P가 직교 → 고유 기저가 직교 (공분산 행렬이 여기 해당)
- 거듭제곱, 행렬 지수가 쉬워지는 이유

**특이값 분해 (SVD)**
- A = UΣVᵀ — 임의의 행렬을 "회전 → 스케일 → 회전"으로 분해
- 고유값 분해의 일반화 (정방행렬이 아니어도 가능)
- 기하학적 의미: 타원체의 주축 찾기

**PCA (주성분 분석)**
- 공분산 행렬의 고유값 분해 = 데이터의 주요 변동 방향 찾기
- 차원 축소: 큰 고유값에 대응하는 축만 남기기
- ML 용도: 특징 추출, 노이즈 제거, 시각화

### 구현 과제

```

         - 2×2 행렬의 고유벡터/고유값을 직접 계산
         - 단위원이 타원으로 변환되는 것을 그려서 고유벡터 = 주축임을 확인

         - 모캡 데이터 또는 3D 포인트 클라우드에 적용
         - 주성분 2~3개만 남겼을 때 복원율 확인

과제 5-3: SVD로 이미지 압축
         - 이미지 행렬에 SVD 적용
         - 상위 k개 특이값만 유지하면서 k를 변화시키며 화질 비교
```

## Phase 6 — ML 연결: 텐서 연산과 미분 (1~2주)

지금까지의 선형대수를 ML 프레임워크의 언어로 번역하는 단계.

### 핵심 개념

**텐서 = 다차원 배열의 선형대수적 해석**
- 스칼라(0차) → 벡터(1차) → 행렬(2차) → 텐서(3차 이상)
- 배치 처리: 여러 입력을 동시에 변환 → 텐서의 축이 하나 추가
- 브로드캐스팅의 수학적 의미

**야코비안 (Jacobian)**
- 다변수 함수의 "국소적 선형 근사"
- 각 출력이 각 입력에 얼마나 민감한가를 행렬로 정리한 것
- 기하학적 의미: 비선형 변환의 국소적 스케일/회전

**체인 룰의 행렬 형태**
- 합성 함수의 야코비안 = 각 단계 야코비안의 곱
- 이것이 신경망 역전파의 수학적 본질
- Forward mode vs Reverse mode autodiff
  - Forward: 야코비안-벡터 곱 (입력 하나에 대한 모든 출력의 변화)
  - Reverse: 벡터-야코비안 곱 (출력 하나에 대한 모든 입력의 변화)
  - 파라미터가 많고 출력(loss)이 하나 → Reverse가 효율적 → 역전파

### 구현 과제

```

         - NumPy만 사용 (PyTorch/TF 금지)
         - 각 층의 야코비안을 명시적으로 계산
         - PyTorch autograd 결과와 비교 검증

         - 삼각형의 꼭짓점 위치를 파라미터로
         - 렌더링 결과와 타겟 이미지의 차이를 loss로
         - loss에 대한 꼭짓점 위치의 그래디언트를 직접 계산
         - 경사 하강법으로 삼각형을 타겟 위치로 이동

         - "프레임워크가 뭘 해주는가"를 체감
```
